## 2. 자료군

### 약국 운영

근무약사 업무·행정·운영 관련 자료. 임상 상담 엔진과 분리하고 향후 관리자 모듈 참고로만 사용한다.

### OTC 상담/실전/핵심

감기, 통증, 위장, 피부, 여성, 소아 등 증상별 실무 자료. 상담카드의 intent 분류, 질문 순서, 환자 친화적 표현 후보로 가치가 높다. 임상 사실은 공식 재검증한다.

### 복약지도/조제실

짧은 시간에 약을 훑고 환자에게 핵심을 전달하는 구조가 유용하다. UI의 `3초 스캔 → 지금 말할 문장 → 단골 질문 → 한마디` 개념에 반영했다. 개별 약물 내용은 별도 검증한다.

### 경구피임제

제품·시점·놓친 정수·포장 위치·상호작용·위험신호를 먼저 확인해야 한다는 슬롯 설계에 반영했다. 구체 누락 규칙은 최신 제품 라벨/공식 지침 없이 사용하지 않는다.

### 증상별 해답 노트 26개

주제가 세분돼 있어 첫 카드 인벤토리와 테스트 matrix를 만드는 데 유용하다. 감기·비염·기침·인후통, 소화/속쓰림/장염, 여성, 소아, 피부 등이 포함된다. 치료 규칙은 모두 candidate 상태다.

### 정답멘트/상담 시나리오/마케팅

현장 표현과 대화 흐름은 참고할 수 있으나, 복합판매·업셀링·강한 제품 권유는 안전 정책에 의해 차단한다. 매출/재고는 임상 점수에 영향을 줄 수 없다.

### 셀메드 이미지 59개

제품·진열·POP·조합 판매·건기식·외용제·일부 동물약 등 마케팅/제품 자료 성격이 강하다. 제품명·카테고리 메타데이터 후보 외에 임상 근거로 사용하지 않는다.

### NQNA/건강체크

외형·생김새·건강 신호 추정 및 제품/영양 연결이 포함돼 있다. 외형 기반 진단/결핍 추정은 runtime에서 제외한다.

### 네이쳐스팜 이미지 9개

비표준 체질/‘기운’ 분류와 제품 연결 자료. clinical runtime에서 제외한다.

### 팜뉴트리션/드럭머거/비타민 D/부작용 XLSX

영양성분·약물-영양 관계·용량·부작용 주제가 있다. 검증 과제 목록에는 유용하지만, 정확한 수치와 인과를 그대로 쓰지 않는다. 별도 건기식 모드와 A/B 근거 검토가 필요하다.

### 서평 DOCX

기능의학·항상성·스트레스 축·부신피로·영양 제품 관련 서술이 포함된다. 임상 상담 근거로 사용하지 않는다.

### 동물약

사람 상담과 섞이지 않도록 별도 도메인으로 차단한다. 향후에도 수의/동물약 전문가 검수와 별도 지식팩이 필요하다.

## 3. 설계에 실제 반영한 것

- 증상별 intent inventory
- 짧고 자연스러운 약국 말투의 필요성
- 다음 질문 한 가지를 우선하는 UX
- 소아/임부/현재약/기간/중증도 슬롯
- 약국 카운터에서 3초 안에 훑는 정보 계층
- 성분 중복과 제품 확인의 중요성
- 판촉성 자료와 임상 근거를 분리하는 source policy
- 사람/건기식/동물/운영 도메인 분리

## 4. 의도적으로 반영하지 않은 것

- 원본 멘트의 장문 복사
- 제품 조합판매 전략
- 외모/체질 기반 건강 판정
- 검증되지 않은 용량·역치·상호작용
- 특정 브랜드 자동 우선 추천
- 첨부 자료 전체를 실시간 RAG에 넣는 구조

## 5. 생산 지식으로 전환하는 법

자료의 문장을 복사하는 대신 다음 순서로 재구축한다.

```text
주제/질문 구조 후보
→ 독립된 claim
→ 공식 출처 확인
→ 약사 검수
→ 짧은 자체 문구
→ golden tests
→ signed knowledge pack
```

이 과정을 통과하지 않은 첨부 자료의 내용은 운영 화면에 나오지 않아야 한다.
