# 02. Source Audit and Trust Policy

## 1. 감사 범위

첨부 ZIP에는 124개 파일이 있었다.

- PDF 52개, 1,730쪽
- JPG/JPEG/PNG 70개
- XLSX 1개
- DOCX 1개
- 압축 해제 기준 약 305MB

검토 방식은 `SOURCE_AUDIT.csv`의 `review_method` 열에 기록했다. 원본 내용은 저작권·기밀 신호 때문에 이 패키지에 복제하지 않았다.

## 2. 자료군별 판정

### 실무 가치가 높은 후보군

- OTC 실전/핵심 가이드
- 증상별 해답 노트
- 복약지도·조제실 워크플로
- 경구피임제 상담 자료

이들은 질문 순서, 상담카드 주제, 환자 친화적 표현을 만드는 데 유용하다. 다만 대부분 practitioner education인 C 등급이다. 용량·금기·상호작용·임부·소아·복용 누락 같은 사실은 공식 A/B 출처로 다시 확인해야 한다.

### 판매·판촉 성격이 강한 후보군

- OTC 상담 시나리오 중 복합판매/업셀링 문구
- 정답조합 교재의 일부 판매 프레임
- 셀메드 이미지 묶음
- 약국 상담 마케팅 교재

임상 추천 순위를 바꾸거나 환자 필요보다 판매를 우선하는 표현은 제거한다. 제품명/진열 메타데이터와 일반적인 대화 UX 아이디어 외에는 운영 임상 근거로 쓰지 않는다.

### 건기식·영양 자료

- 팜뉴트리션 자료
- 드럭머거 표
- 비타민 D 이미지
- 영양소 부작용 XLSX

주제 인벤토리와 검증 과제에는 쓸 수 있으나, 간략화된 용량·역치·부작용·상호작용을 그대로 상담에 사용하지 않는다. 건기식 모드에서 claim 단위로 공식 기준과 고품질 임상 근거를 연결한 뒤 약사가 승인해야 한다.

### 제외군

- NQNA의 외형/건강신호 추정
- 네이쳐스팜의 비표준 체질/기운 분류
- 기능의학·부신피로 등 서평 기반 주장
- 사람 모드의 동물약 자료

이 자료들은 clinical runtime에서 완전히 배제하거나 별도 도메인으로 격리한다.

## 3. 신뢰 등급

### A — authoritative current

대한민국의 최신 공식 제품 허가사항, 안전성 공지, DUR 데이터, 적용 범위가 명확한 공공 임상 기준. 고위험 제품별 규칙의 필수 근거다.

### B — high-quality clinical evidence

최신 peer-reviewed guideline, systematic review, 전문학회 합의 등. 제품별 공식 라벨과 충돌하지 않는 범위에서 보조한다.

### C — practitioner education

현장 교육자료, 실무 가이드, 강의자료. 질문 순서·표현·카드 후보를 제공하지만 임상 사실로 자동 승격되지 않는다.

### D — marketing/product material

제품 판촉, 조합판매, 진열·POP, 마케팅 콘텐츠. 임상 근거로 사용하지 않는다.

### X — excluded

비표준 진단 방식, 잘못된 도메인, 운영 자료, 검증 불가 자료. 사람 임상 런타임에 절대 노출하지 않는다.

`SOURCE_AUDIT.csv`의 `C-D`는 파일 안에 실무 교육과 판매 프레임이 혼재한다는 감사용 표기다. 정식 SourceRecord/Claim enum으로 저장하지 말고, 반입 시 claim별로 C 또는 D로 분해한다. 분해가 불가능하면 더 제한적인 D를 적용한다.

## 4. Claim 단위 원칙

파일에 한 번 등급을 붙이는 것으로 충분하지 않다. 같은 파일에도 좋은 상담 표현과 위험한 용량표가 함께 있을 수 있으므로 최종 판단 단위는 `Claim`이다.

각 claim은 다음을 가져야 한다.

- 고유 ID와 claim type
- subject/predicate/object/qualifier
- 위험도
- 원 출처와 정확한 locator
- 서로 충돌하는 claim ID
- 공식 출처 확인 여부
- 약사 승인 여부
- 검증일과 만료일
- reject/retire 사유

고위험 claim은 A 출처가 없으면 원칙적으로 publish하지 않는다. B로만 가능한 예외는 명시적 약사 판정과 사유를 남긴다.

## 5. 충돌 처리

- 서로 다른 수치를 평균하지 않는다.
- 더 최근이라는 이유만으로 출처 성격을 무시하지 않는다.
- 제품별 공식 라벨이 일반 교육자료보다 우선한다.
- 지역/제품/제형/연령/임신 상태가 다른 규칙을 하나로 합치지 않는다.
- 충돌이 해소될 때까지 카드를 차단한다.
- 모델이 “상식적으로” 하나를 선택하게 하지 않는다.

## 6. 저작권·기밀

폴더명과 여러 문서에 외부 반출·복제 제한을 시사하는 문구가 있고, 상업 교육자료로 보이는 파일이 다수다. 따라서:

- 원본 파일·페이지 이미지·장문 원문을 GitHub에 올리지 않는다.
- 원문을 runtime prompt에 통째로 넣지 않는다.
- 상담카드는 사실과 아이디어를 독립적으로 재작성하고, 필요한 경우 사용권을 확인한다.
- `SOURCE_AUDIT.csv`에는 해시와 내부 locator만 두고 파일 자체를 배포하지 않는다.
- 출처 사용권이 불명확하면 내부 검수 참고로만 쓰고 production pack에서 제외한다.

## 7. 자료 반입 절차

1. 소유·사용권 확인
2. 해시·버전·날짜 등록
3. 도메인 분류
4. trust tier 예비 판정
5. 자동 추출은 candidate 상태로만 저장
6. 원문 표현/판촉/과장을 별도 태깅
7. 공식 출처 대조
8. 약사 검수
9. schema/lint/golden test
10. immutable pack publish

## 8. 운영 차단 규칙

다음 중 하나면 production build를 실패시킨다.

- source tier C/D/X만 참조하는 고위험 claim
- locator 없는 claim
- `verified_at` 또는 `expires_at` 없는 claim
- reviewer가 없는 publish 카드
- 만료/충돌/rejected claim 참조
- `draft`, `placeholder`, `synthetic` 예시의 포함
- 사람 모드에 animal/supplement_nonstandard source 포함
- 매출·마진·재고 소진을 추천 점수로 사용하는 필드

## 9. 파일별 결과

`SOURCE_AUDIT.csv`는 124개 파일 모두에 대해 다음을 제공한다.

- 파일 ID, 경로, SHA-256, 형식, 페이지/텍스트량
- 자료군·도메인·신뢰 등급
- 런타임 정책
- 허용 용도와 금지 용도
- 임상 재검증 필요성
- 검토 방법과 메모

이 표는 clinical evidence registry가 아니라 **원본 자료의 사용 경계 감사표**다.
